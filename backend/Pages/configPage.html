Seerr Integration

Enable Seerr integration for all users. Moonfin handles authentication through a server-side proxy so users can sign in once and stay authenticated across clients.
Where Jellyfin can reach your Seerr server. Start the address with http:// or https://. Examples:
  • http://192.168.1.10:5055 (another machine on your network)
  • http://seerr:5055 (Docker, on the same network as Jellyfin)
  • https://requests.example.com (a public address)
This is the address the Jellyfin server connects to behind the scenes, not the one you open in a web browser, so a local network or Docker address works fine. You do not enter a username, password, or API key here; people sign in later from the app.

Advanced: If a reverse proxy in front of Seerr shows its own login page, let the Jellyfin server through it. If Jellyfin itself sits behind a reverse proxy, make sure that proxy forwards /Moonfin/ requests to Jellyfin.
Optional override. The name shown in the UI (e.g., "Requests", "Media Requests"). Leave blank to auto-detect from the configured URL. The icon is always auto-detected regardless of this setting.
The address Seerr uses to reach this Jellyfin server for request notifications (the webhook Seerr calls back). Set this when Seerr and Jellyfin run in separate containers or hosts, since the server otherwise can only guess a local address that Seerr may not be able to reach. Use the base URL, for example http://192.168.1.10:8096 or https://jellyfin.example.com.

MDBList Integration

Provide a server-wide MDBList API key that all users can use for ratings (IMDb, Rotten Tomatoes, Metacritic, etc.). Users who set their own key in their Moonfin client settings will use that instead.

TMDB Integration

Provide a server-wide TMDB API key that all users can use for episode ratings. Users who set their own key in their Moonfin client settings will use that instead. Supports both v3 API keys and v4 Bearer tokens.

Retro Games (EmulatorJS)

Lets Moonfin clients browse and play ROMs from a "Mixed Content" library using EmulatorJS. Organize ROMs as System/<game>/rom.ext with any BIOS files at the top of each system folder. No setup is needed for the emulator itself: cores stream from the EmulatorJS CDN by default. For offline use, drop the EmulatorJS data/ folder into <plugin data>/emulatorjs/data/ and it is used automatically.
Online (default): cores stream from the EmulatorJS CDN. Zero setup, but each client needs internet access when a game launches.
Offline (self-host): install the cores on this server once so clients play with no internet and lower latency. Use either option below; clients switch to the local copy automatically once it is installed.

Default User Settings

Configure default Moonfin settings for all users. New users and users who haven't customized a setting will see these values. Users can always override defaults in their own Moonfin settings panel.
How it works: These are the lowest-priority fallback. Resolution order: User's device profile → User's global settings → These admin defaults → Built-in defaults.
Important: New nullable settings use Not set / Yes / No style inputs. Not set means "do not force this value" so users keep their own choice.

Web Plugin Status

Moonfin uses the File Transformation plugin to inject a header button in Jellyfin Web.

What it does: the injected button opens the Flutter app at /Moonfin/Web/.

Setup:

  1. Install the File Transformation plugin (add https://www.iamparadox.dev/jellyfin/plugins/manifest.json to your plugin repositories)
  2. Restart Jellyfin
  3. Force refresh your browser (Ctrl+Shift+R)

UI not loading? Go to Dashboard > Scheduled Tasks and run the Moonfin Startup task once, then refresh your browser.

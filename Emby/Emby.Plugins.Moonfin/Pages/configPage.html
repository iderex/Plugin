Moonfin Web Client

Pre-fills the Add Server dialog in Moonfin web. Leave empty to auto-fill from the current Emby origin. Use the Emby base URL (for example, http://192.168.1.10:8096), not /web.
Forces Moonfin web to auto-connect to this URL on load. Leave empty to use plugin-mode auto-connect behavior. Use the Emby base URL and do not include a /web suffix.
Allows Moonfin web to scan private LAN ranges via WebRTC hints. Disable if your browser environment blocks or restricts WebRTC.

Seerr Integration

Enable Seerr integration for all users. Moonfin handles authentication through a server-side proxy so users can sign in once and stay authenticated across clients.
Where Emby can reach your Seerr server. Start the address with http:// or https://. Examples:
  • http://192.168.1.10:5055 (another machine on your network)
  • http://seerr:5055 (Docker, on the same network as Emby)
  • https://requests.example.com (a public address)
This is the address the Emby server connects to behind the scenes, not the one you open in a web browser, so a local network or Docker address works fine. You do not enter a username, password, or API key here; people sign in later from the app.

Advanced: If a reverse proxy in front of Seerr shows its own login page, let the Emby server through it. If Emby itself sits behind a reverse proxy, make sure that proxy forwards /Moonfin/ requests to Emby.
Optional override. The name shown in the UI (e.g., "Requests", "Media Requests"). Leave blank to auto-detect from the configured URL. The icon is always auto-detected regardless of this setting.
SSO Proxy: When Seerr is enabled, Moonfin provides single sign-on. Users authenticate once with their Emby credentials, and the session is stored server-side. All Moonfin clients access Seerr API through /Moonfin/Seerr/Api/ without re-authenticating.

MDBList Integration

Provide a server-wide MDBList API key that all users can use for ratings (IMDb, Rotten Tomatoes, Metacritic, etc.). Users who set their own key in their Moonfin client settings will use that instead. This key also powers the MDBList official list home rows (IMDb Top 250, Most Popular, and similar charts), which the "Moonfin MDBList Official Lists Sync" scheduled task fetches and caches on the server.
Fetches IMDb charts (Top 250 movies/TV, Most Popular, and similar) on a schedule so clients can show them as home rows and custom rows can resolve the IMDb source. No API key required.

TMDB Integration

Provide a server-wide TMDB API key that all users can use for episode ratings. Users who set their own key in their Moonfin client settings will use that instead. Supports both v3 API keys and v4 Bearer tokens. This key is also used to fetch posters for the MDBList official list home rows. Without it, those rows still work but show titles without artwork.
Pre-warms the studio (production company) logo cache from TMDB on a schedule so clients can show studio logos on the details screen. Uses the server-wide TMDB key above.

Optional. Paste a Firebase service-account JSON to send Seerr push notifications directly through FCM from this server. Leave empty to use the hosted Moonfin push relay instead. Secret: never logged.
Optional path to a service-account JSON file on the server. When both this and the JSON above are set, the file path wins.

Retro Games (EmulatorJS)

Lets Moonfin clients browse and play ROMs from a "Mixed Content" library using EmulatorJS. Organize ROMs as System/<game>/rom.ext with any BIOS files at the top of each system folder. No setup is needed for the emulator itself: cores stream from the EmulatorJS CDN by default.
Online (default): cores stream from the EmulatorJS CDN. Zero setup, but each client needs internet access when a game launches.
Offline (self-host): install the cores on this server once so clients play with no internet and lower latency. Use either option below; clients switch to the local copy automatically once it is installed.

Upload an EmulatorJS data folder zipped up (the zip must contain loader.js somewhere inside). Get it from EmulatorJS releases (extract the archive and zip its data folder). ~290 MB.
Downloads the zip in the background. Leave the URL blank to use an emulatorjs-data.zip asset on the plugin's GitHub release. Click Save below to persist the URL.
Select the libraries that hold your game ROMs, then click Save below. If none are selected, libraries whose name contains "game", "rom", or "emulat" are auto-detected.
Box art is fetched automatically from the keyless libretro thumbnail server (matched by game name); no API keys or accounts are needed.

Default User Settings

Configure default Moonfin settings for all users. New users and users who haven't customized a setting will see these values. Users can always override defaults in their own Moonfin settings panel.
How it works: These are the lowest-priority fallback. Resolution order: User's device profile → User's global settings → These admin defaults → Built-in defaults.
Important: New nullable settings use Not set / Yes / No style inputs. Not set means "do not force this value" so users keep their own choice.

Push Defaults To Existing Users

Apply the configured defaults above to every existing user now. Only fields that are set (not "Not set") will be merged into user global settings. Save your default settings first if you made changes.
Warning: This action updates existing user profiles immediately for all set fields. Device-specific profile overrides are not touched.

Broadcast Message

Send a live message to all currently connected Moonfin clients. This message is not saved and only appears for users with an active connection.

Opening Moonfin

Open Moonfin directly at /Moonfin/Web/ on your Emby server.

Example: http://192.168.1.10:8096/Moonfin/Web/

Note: The header-button injection available in the Jellyfin version is not available in the Emby version. Navigate to the URL above directly.
